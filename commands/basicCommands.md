# Основные команды Linux / MacOS

## Работа с путями и директориями

1. `ls` - показать содержимое текущей директории
    - `-a`, `--all` - показать всё, в том числе скрытые файлы и папки (название которых начинается с точки, например, папка `.git` в корне репозитория)
    - `-l`, `--long` - длинный формат (не только имя файла, но и информация о нём)
    - `-h` - имеет смысл вместе с флагом `-l`: показывает размер файла в удобочитаемом (**h**uman-readable) формате. Флаги можно совмещать и располагать в любом
      порядке: `ls --all -l -h`, `ls -a -l -h`, `ls -alh`, `ls -hla` - эквивалентные команды.
1. `pwd` - показать путь до текущей директории от корня файловой системы
1. `cd <имя папки>` (от **c**hange **d**irectory) - сменить текущую директорию.
    * Специальные обозначения: `.` - текущая директория, `..` - родительская. `~` - домашняя директория текущего пользователя (
      синоним `/home/<имя_пользователя>`).
    * Путь может быть
        * абсолютным (начинается со слэша `/`, например, `/home/valerii_zuev/Documents/stu`)
        * относительным (из папки `/home/valerii_zuev/Downloads/abc` тот же путь может быть написан как `../../Documents/stu`)
        * написанным относительно домашней директории (`~/Documents/stu`)
1. `wget <URL>` - скачать файл по ссылке

## Системные ресурсы

1. `htop` - показать загрузку ЦП по ядрам и оперативной памяти, а также активные процессы. Выйти - `Ctrl+C` (на маке `System+C`)
1. `du` - показать, сколько места на диске занимают папки в текущей директории

## Git

1. `git init` - создаёт репозиторий в данной папке (появляется папка `.git`, в которой по мере накопления коммитов будут складываться служебные файлы Git)
2. `git status` - посмотреть, что у нас творится в репозитории. Можно передать аргумент - имя папки, в которой смотрим изменения, тогда всё остальное будет проигнорировано. Например, `git status .`
3. `git log` - посмотреть логи коммитов (что когда закоммитили и с каким сообщением). Я люблю так форматировать вывод: `git log --pretty=oneline --abbrev-commit -n 10`, что выводит 10 последних коммитов, каждый в отдельной строчке, хэш коммита показан в сокращённом виде.
4. `git add <directory or file>` - добавить файл (или папку с файлами, пустую папку не получится) в набор файлов, которые попадут в коммит. Отображаются зелёным в `git status`
5. `git commit` - сделать коммит, то есть зафиксировать изменения (не все, а только добавленные ранее через `git add`) в памяти Git'a. Открывается текстовый редактор, надо написать сообщение к коммиту, закрыть редактор и выйти из него. Чтобы не использовать редактор, можно добавить флаг `-m "<commit message>"`.
6. `git fetch` - если у репозитории есть удалённое хранилище (например, на GitHub), сходить узнать, нет ли чего нового
7. `git pull` - обновить текущую ветку, если в удалённом хранилище появились коммиты, которых в локальном ещё нет
8. `git push` - выгрузить локальные коммиты из текущей ветки в удалённое хранилище
9. `git branch` - посмотреть, какие есть ветки и на какой ветке мы сейчас
10. `git switch <branch name>` - переместиться в ветку с указанным именем
11. `git checkout <commit hash>` - перейти к той версии репозитория, которая была после коммита с указанным хэшом. Работает, если при этом в репозитории не были изменены (по сравнению с последним коммитом) файлы, которые меняются при  переходе к указанному коммиту, иначе выдаёт ошибку. Например, если в репозитории два
12. `git clone <URL>` - скачать себе репозиторий локально по ссылке. В папке, в которой мы сейчас находимся, появится подпапка, в которой лежат файлы репозитория. 

# Полезные советы

1. `tab` после частично набранного имени команды (или файла/папки) допишет слово за вас (если возможен только один вариант). Если возможны несколько вариантов,
   начинающиеся с одного префикса, будет вставлен этот префикс.
1. `Ctrl+R` (на Маке `System+R`) - поиск по недавним командам
1. `man <имя команды>` (от **man**ual) - справка по команде. Также часто доступна команда `<имя команды> --help` и/или `<имя команды> -h`
1. В консоли можно заводить переменные, создавать временные (живущие только до того, как мы закроем терминал) и постоянные функции (доступные в каждом
   терминале, записываются в `~/.bashrc`), писать циклы и условия. Но это не совсем тема данного
